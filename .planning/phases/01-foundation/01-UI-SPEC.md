# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation.
> Covers Auth screens (Register, Login, Password Reset), i18n scaffold (DE + EN), and the app shell / navigation structure.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | Tamagui 2.0 (`@tamagui/core`, `@tamagui/config/v5`) | STACK.md — locked, no deviations |
| Preset | `defaultConfig` from `@tamagui/config/v5`, extended in `tamagui.config.ts` | STACK.md |
| Component library | Tamagui built-ins (`Input`, `Button`, `Stack`, `YStack`, `XStack`, `Text`, `Form`, `Sheet`, `Dialog`) | STACK.md |
| Icon library | `@expo/vector-icons` — Feather icon set | Default; confirmed available in Expo SDK 56 |
| Font | Inter (via `@tamagui/font-inter`, loaded by `expo-font` on native) | STACK.md |
| Platforms | iOS, Android, Web (universal — single token set) | REQUIREMENTS.md |
| shadcn gate | Not applicable — Expo/React Native app, not Next.js/Vite | — |
| Registry safety | Not applicable — no shadcn registry | — |

**Token implementation rule:** All design values below MUST be expressed as Tamagui compile-time tokens in `tamagui.config.ts`. No inline style props with raw pixel values on any component. Use `$space.md`, `$size.icon`, `$color.accent` etc. Violation of this rule defeats the Tamagui compiler optimisation.

## Spacing Scale

Declared in `tamagui.config.ts` under `tokens.space`. All values are multiples of 4.

| Token name | Value | Tamagui token key | Usage |
|------------|-------|-------------------|-------|
| xs | 4px | `$space.xs` | Icon-to-label gaps, badge padding, divider insets |
| sm | 8px | `$space.sm` | Compact internal padding (input icon offset, label-to-field gap) |
| md | 16px | `$space.md` | Default element spacing, form field vertical gap, card internal padding |
| lg | 24px | `$space.lg` | Section padding, auth card internal padding (sides) |
| xl | 32px | `$space.xl` | Layout vertical rhythm between form sections |
| 2xl | 48px | `$space.2xl` | Top padding above logo on auth screens, major section breaks |
| 3xl | 64px | `$space.3xl` | Safe-area-aware bottom padding on web, page-level vertical breathing room |

**Exceptions:**

| Exception | Value | Reason |
|-----------|-------|--------|
| Minimum touch target (buttons, links) | 44px height | iOS HIG + Android accessibility — applies to all tappable elements |
| Tab bar item touch target | 48px height | Platform tab bar minimum for comfortable thumb reach |
| Form field height | 52px | Comfortable text entry on mobile; consistent with Tamagui `$size.5` |
| Logo mark container | 80px × 80px | Auth screen centrepiece; sized for recognisability at first launch |

## Typography

Font: Inter. Loaded via `expo-font` + `@tamagui/font-inter` on native. CSS variable `"Inter", Helvetica, sans-serif` on web.

All sizes and weights declared in `tamagui.config.ts` under `tokens.font` (heading font + body font, both Inter).

| Role | Size | Weight | Line Height | Tamagui usage |
|------|------|--------|-------------|---------------|
| Body | 16px | 400 (regular) | 1.5 | Form helper text, list descriptions, paragraph copy |
| Label | 14px | 400 (regular) | 1.4 | Input labels, tab bar labels, secondary metadata |
| Heading | 22px | 600 (semibold) | 1.2 | Screen titles ("Registrieren", "Anmelden"), card headings |
| Display | 28px | 600 (semibold) | 1.1 | App name / logo wordmark on auth screens only |

**Rules:**
- Maximum 4 sizes in use across Phase 1. No ad-hoc sizes outside this table.
- Maximum 2 weights in runtime use: 400 (regular) and 600 (semibold). No other weights are permitted anywhere in Phase 1.
- Do not use font sizes below 14px anywhere — minimum for accessibility across all platforms.
- German strings are 20–30% longer than English equivalents. All text containers must use dynamic width (`flex: 1` or `flexWrap: 'wrap'`). No fixed-pixel-width text boxes.
- Letter spacing: 0 for body and label; -0.3px for heading; -0.5px for display.

## Color

Palette: warm, literary — inspired by aged paper, ink, and warm wood. Appropriate for a book club app. Light theme is the Phase 1 default; dark theme tokens are declared but Phase 1 screens render light only.

All hex values below are declared as named tokens in `tamagui.config.ts` under `themes.light` and `themes.dark`.

### Light Theme (Phase 1 default)

| Role | Hex | Tamagui token | Usage |
|------|-----|---------------|-------|
| Dominant (60%) — background | `#FDFAF6` | `$background` | Screen background, auth card background, tab bar background |
| Secondary (30%) — surface | `#F2EDE4` | `$backgroundStrong` | Input field fill, secondary cards, section backgrounds |
| Accent (10%) — brand | `#7C4B2A` | `$accent` | **Reserved for:** primary button fill, active tab indicator, focus ring on inputs, logo mark background |
| Text primary | `#1A1209` | `$color` | All body copy, headings, form values |
| Text secondary | `#6B5C47` | `$colorSecondary` | Placeholder text, helper text, labels, disabled state copy |
| Border | `#D6CBBC` | `$borderColor` | Input field borders, card outlines, dividers |
| Destructive | `#B33A3A` | `$destructive` | Delete account action (AUTH-05 logout is not destructive — no confirmation needed), error state borders |
| Success | `#2E7D5A` | `$success` | Email-sent confirmation banner on password reset screen |
| White | `#FFFFFF` | `$backgroundPress` | Button label on filled accent buttons, overlay surfaces |

### Dark Theme (declared, not used in Phase 1)

Declare dark theme tokens in `tamagui.config.ts` so the light/dark toggle in Phase 5 requires no token additions. Minimum token set:

| Token | Dark value |
|-------|------------|
| `$background` | `#1A1209` |
| `$backgroundStrong` | `#2A1F14` |
| `$accent` | `#C47A3D` |
| `$color` | `#F2EDE4` |
| `$colorSecondary` | `#9E8A74` |
| `$borderColor` | `#3D2E20` |
| `$destructive` | `#E05555` |
| `$success` | `#4AB07A` |

**Accent reserved for (exhaustive list — do not extend without updating this spec):**
1. Primary button background (`Button` variant `primary`)
2. Active tab indicator dot / underline
3. Input focus ring / border when focused
4. Logo mark background fill
5. Checkbox / radio selected state (future phases)

**Destructive reserved for:**
- Delete account action only in Phase 1 (appears on profile screen, Phase 2+). In Phase 1, the only use is on password reset error states (invalid token banner border).

## Responsive Breakpoints

Declared in `tamagui.config.ts` under `media`. Used via Tamagui responsive props (`$sm`, `$md`, `$gtMd`).

| Token | Condition | Layout behaviour |
|-------|-----------|-----------------|
| `$sm` | `maxWidth: 390` | Single-column, full-width inputs, stacked layout (default mobile) |
| `$md` | `maxWidth: 768` | Single-column with horizontal padding `$space.xl` each side |
| `$gtSm` | `minWidth: 391` | Auth card becomes a centred card (`maxWidth: 420px`) floating on the surface |
| `$gtMd` | `minWidth: 769` | Auth card: `maxWidth: 440px`, vertical centering with visible background texture |

**Auth screen web layout:** On `$gtSm` breakpoint and above, auth screens render as a centred card (`maxWidth: 420px`, `borderRadius: 16px`, `padding: $space.lg`, box shadow `0 4px 24px rgba(28,16,4,0.10)`) floating on the `$background` surface. Below `$gtSm` (phones), the card fills the full screen with no border radius.

## i18n Visual Contract

**Scope for Phase 1:** `auth` namespace (all auth screens) + `common` namespace (shared labels, CTAs, errors) + `nav` namespace (tab labels, header titles).

**Text expansion budget:**

| Namespace | Max expansion over English | Layout mitigation |
|-----------|---------------------------|-------------------|
| `auth.*` | +35% (German is verbose) | All text elements use `flexWrap: 'wrap'` and `numberOfLines` is NOT set |
| `common.cta.*` | +25% | Buttons use `flexShrink: 1` and allow label to wrap to 2 lines on narrow screens |
| `nav.*` | +20% | Tab labels allow 2 lines (14px); web nav items use `white-space: nowrap` — abbreviate if over 12 chars |

**RTL readiness:** German and English are both LTR. RTL is NOT required for Phase 1. However:
- Do NOT hard-code `textAlign: 'left'` — use `textAlign: 'start'` where text alignment must follow reading direction.
- Do NOT use `marginLeft`/`marginRight` for icon-to-text offsets — use `gap` or `marginStart`/`marginEnd`.
- This ensures Phase 5+ can add RTL languages without rework.

**SSR / hydration safety (web):**

- All i18n `t()` calls that render on the server-rendered HTML shell MUST use `useDidFinishSSR()` guard from `@tamagui/use-did-finish-ssr`.
- Server-safe default: render English copy on server; hydrate with device locale on client.
- Pattern:
  ```tsx
  const isClient = useDidFinishSSR()
  return <Text>{isClient ? t('auth.signIn') : 'Sign In'}</Text>
  ```
- Applies to: all auth screen headings, CTA labels, and any text visible above-the-fold on web.

**Plural forms — Phase 1 scope:**
- Phase 1 has no countable UI strings. Plural contract deferred to Phase 2 (member counts) and Phase 4 (vote counts).
- Declare i18next `pluralSeparator: '_'` and `compatibilityJSON: 'v4'` in `lib/i18n/index.ts` now so plural keys work without config changes later.

## Motion / Animation Contract

Animation driver: platform-conditional as required by PITFALLS.md Pitfall 10.

```typescript
// In tamagui.config.ts
import { isWeb } from 'tamagui'
import { animations as animationsCSS } from '@tamagui/animations-css'
import { animations as animationsReanimated } from '@tamagui/animations-react-native'

animations: isWeb ? animationsCSS : animationsReanimated
```

**Animation presets** — declared in `tamagui.config.ts`:

| Preset name | Duration | Easing / Spring | Used for |
|-------------|----------|-----------------|----------|
| `fast` | 120ms | Spring: damping 20, stiffness 280 | Button press feedback, input focus border transition |
| `medium` | 220ms | Spring: damping 18, stiffness 200 | Screen-level transitions, banner appear/disappear |
| `slow` | 350ms | Spring: damping 15, stiffness 120 | Sheet/Dialog open (Phase 2+), not used in Phase 1 |

**Phase 1 animation inventory:**

| Element | Animation | Preset |
|---------|-----------|--------|
| Button press | Scale 0.98 + opacity 0.85 via `pressStyle` | `fast` |
| Input focus | Border color transition `$borderColor` → `$accent` | `fast` |
| Error banner appear | Opacity 0 → 1 + translateY 4px → 0 | `medium` |
| Success state (register/reset) | Opacity 0 → 1 | `medium` |
| Tab bar active indicator | Color crossfade | `fast` |

**Rules:**
- No layout-shift animations on auth screens (form reflows must be instant, not animated).
- No looping / idle animations in Phase 1 — reserve for vote leaderboard in Phase 4.
- Respect `reduceMotion` preference: wrap all `enterStyle` / `exitStyle` animations with `useReducedMotion()` guard; fall back to instant opacity change when `reduceMotion = true`.
- Tamagui's `disableExtraction: true` is set in development (Babel plugin option) to preserve hot-reload speed; animations still run at runtime in dev.

## Tamagui Config File Contract

The executor MUST create `tamagui.config.ts` in the `expo-app/` root with exactly the following structure (tokens filled from this spec):

```typescript
import { createFont, createTamagui, createTokens, isWeb } from '@tamagui/core'
import { defaultConfig } from '@tamagui/config/v5'
import { animations as animationsCSS } from '@tamagui/animations-css'
import { animations as animationsReanimated } from '@tamagui/animations-react-native'

const config = createTamagui({
  ...defaultConfig,

  // 1. Platform-conditional animation driver (PITFALLS.md Pitfall 10)
  animations: isWeb ? animationsCSS : animationsReanimated,

  // 2. Animation presets
  // fast / medium / slow — see Motion contract above

  // 3. Fonts — Inter for heading + body
  // @tamagui/font-inter createFont() for both roles

  // 4. Tokens — spacing, color, size from this spec

  // 5. Themes — light + dark (both declared, light is default)

  // 6. Media breakpoints — sm / md / gtSm / gtMd from this spec
})

type AppConfig = typeof config
declare module '@tamagui/core' {
  interface TamaguiCustomConfig extends AppConfig {}
}

export default config
```

**Import path rule:** `@tamagui/config/v5` — NOT `@tamagui/config/v4` and NOT plain `@tamagui/config`. Wrong path silently no-ops (STACK.md).
