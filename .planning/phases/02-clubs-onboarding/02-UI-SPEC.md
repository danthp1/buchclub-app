# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Clubs & Onboarding.
> Covers the onboarding flow (username → avatar → create/join), club creation wizard, join-by-code/link flow, public club browse, club detail with member management, profile editing, and the active-club context banner on all tabs.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Design System Alignment Note

**IMPORTANT — Design System Gap from Phase 1:**

Phase 1 was implemented with Inter font and a warm brown token palette (`accent: #7C4B2A`). The authoritative design source (`design/design.md` + all 13 screen reference PNGs) specifies a different system:

- **Fonts:** Archivo Narrow Bold (headlines/display) + IBM Plex Sans Regular/SemiBold (body/UI)
- **Primary button:** Ink Black fill (`#0D0D0D`), NOT warm brown
- **Accent actions:** Electric Blue (`#1A4FE0`) for CTAs/links
- **Background:** `#F0EDE4` (Papier), NOT `#FDFAF6`

Phase 2 adopts the authoritative `design/design.md` system. The executor **must** update `tamagui.config.ts` in Phase 2 to migrate tokens to the correct palette and add the two custom fonts. Existing auth screens (Phase 1) must also be updated for visual consistency — treating this as part of Phase 2 Plan 01.

The Phase 2 token table below is the corrected source of truth for all tokens forward.

## Typography

Phase 2 introduces Archivo Narrow Bold and IBM Plex Sans, replacing Inter from Phase 1. The `tamagui.config.ts` font registration must be updated.

**Font loading:**
- `ArchivoNarrow_700Bold` — loaded via `expo-font` + `@expo-google-fonts/archivo-narrow`
- `IBMPlexSans_400Regular` + `IBMPlexSans_600SemiBold` — loaded via `expo-font` + `@expo-google-fonts/ibm-plex-sans`

All fonts declared in `tamagui.config.ts` under `tokens.font` (headingFont using Archivo Narrow, bodyFont using IBM Plex Sans).

| Role | Font family | Size | Weight | Line height | Tamagui usage |
|------|-------------|------|--------|-------------|---------------|
| Display | Archivo Narrow Bold | 32px | 700 | 36px (1.125) | Screen-level hero titles ("Willkommen", club name on success screen) |
| H1 | Archivo Narrow Bold | 24px | 700 | 28px (1.167) | Screen headers, card titles, book names |
| H2 | Archivo Narrow Bold | 18px | 700 | 24px (1.333) | Section headings within screens |
| Body | IBM Plex Sans Regular | 15px | 400 | 22px (1.467) | Descriptions, body text, club descriptions |
| Body-sm | IBM Plex Sans Regular | 13px | 400 | 20px (1.538) | Captions, meta text, timestamps, member count |
| Label | IBM Plex Sans SemiBold | 12px | 600 | 16px (1.333) | Tags, status chips, category labels |
| Button | IBM Plex Sans SemiBold | 15px | 600 | 20px (1.333) | All CTA button labels |

**Rules:**
- Archivo Narrow reserved exclusively for: display headings, screen titles, hero elements, club names as primary content. Never in forms, input labels, body text.
- IBM Plex Sans for all interactive and body content: inputs, labels, button labels, descriptions, metadata.
- Maximum 6 sizes in use across Phase 2 (display 32, h1 24, h2 18, body 15, body-sm 13, label 12). No ad-hoc sizes outside this table.
- Maximum 2 weights: 400 (Regular) + 600 (SemiBold) for IBM Plex Sans; 700 (Bold) for Archivo Narrow.
- Minimum font size: 12px (label tokens only). Never below 12px anywhere.
- German string expansion: all text containers use `flexWrap: 'wrap'`, no fixed-pixel-width text boxes.

### Auth Guard Extension (`app/_layout.tsx` `InitialLayout`)

The existing auth guard is extended with a second gate:

```
if (!initialized) return (splash)
if (!session)     → redirect to (auth)/sign-in
if (!onboardingCompleted) → redirect to (onboarding)/username
                          → redirect to (app)/clubs
```

The `onboardingCompleted` flag lives in `store/club.store.ts` (new), persisted via `expo-sqlite`.

### Deep Link Extension

The existing deep link handler in `app/_layout.tsx` is extended:

```
buchclub://join?code=XXXXXXXX  → (app)/clubs/join
buchclub://join/XXXXXXXX       → (app)/clubs/join  (club invite via short URL)
```

During onboarding, deep link intercept: if `!onboardingCompleted` and invite link arrives → complete username first, then auto-navigate to join screen with code pre-filled.

## Interaction Patterns

### Onboarding Flow

```
(auth)/sign-in → auth success
  ↓ InitialLayout checks onboardingCompleted (Zustand)
  ↓ false
(onboarding)/username
  ↓ valid username submitted
(onboarding)/avatar
  ↓ avatar selected or skipped
  ↓ Sheet opens (create-or-join — 35% height, not dismissable)
  ↓ user taps "Club erstellen"
(app)/clubs/create (wizard)
  ↓ step 4 confirmed
(app)/clubs/create/success
  ↓ "Zum Club"
(app)/clubs/[id]
  ↓ onboardingCompleted = true (stored in expo-sqlite)
```

### Wizard Navigation

- Forward: `<Button primary "Weiter">` — validates current step before advancing
- Back: hardware back / Feather back arrow — no validation, preserves entered data
- Cancel (X): Dialog confirm "Wizard abbrechen?" / "Abandon setup?" → discard and go to clubs list
  - Exception: during onboarding, cancel is not shown (flow is mandatory)
- Step indicator: tapping past steps navigates back (forward-only is blocked by validation)

### Bottom Sheet (Tamagui Sheet)

```typescript
// Interaction contract for all Sheets in Phase 2
<Sheet
  modal={true}
  open={open}
  onOpenChange={setOpen}
  snapPoints={[35]}        // or [50] for invite sheet
  dismissOnSnapToBottom={true}   // except create-or-join: dismissOnSnapToBottom={false}
  animation="slow"               // 350ms spring preset from tamagui.config.ts
>
  <Sheet.Overlay animation="medium" enterStyle={{ opacity: 0 }} exitStyle={{ opacity: 0 }} />
  <Sheet.Handle />
  <Sheet.Frame borderTopLeftRadius={24} borderTopRightRadius={24}>
    {/* content */}
  </Sheet.Frame>
</Sheet>
```

### Deep Link Entry (Join Flow)

```
buchclub://join?code=XXXXXXXX arrives
  ↓ _layout.tsx Linking.useURL() fires
  ↓ if !onboardingCompleted → store code in Zustand, complete onboarding first
    → after onboarding complete → auto-navigate to clubs/join with code pre-filled
  ↓ if onboardingCompleted → navigate to (app)/clubs/join?code=XXXXXXXX
    → CodeInput auto-fills, validation runs automatically
```

### Club Switcher

```
ClubBanner (top of each tab) → tap Feather 'chevron-down'
  → Sheet opens (snapPoints: [55])
  → List of user's clubs as ClubCard (compact variant, no Join button)
  → Tap club: sets activeClubId in Zustand + expo-sqlite, sheet closes
  → All tabs update to new club context immediately
```

### Keyboard Avoidance

All screens with inputs use `KeyboardAvoidingView` (behaviour: `'padding'` on iOS, `'height'` on Android). The bottom Sheet uses Tamagui's built-in `Sheet.ScrollView` for keyboard avoidance inside sheets.
