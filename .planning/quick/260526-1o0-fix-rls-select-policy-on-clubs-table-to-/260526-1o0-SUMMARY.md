# Summary: Fix RLS SELECT policy on clubs

## What was done
Applied migration `fix_clubs_select_policy_creator` to add `created_by = auth.uid()` as a third OR condition to `clubs_select_public_or_member`.

## Root cause
`.insert().select().single()` on a private club returned null because the SELECT policy required `is_public OR EXISTS(member)` — but the `club_members` row is inserted in a separate statement *after* the club, so the creator wasn't a member yet when the chained SELECT ran.

## Fix
Policy now: `is_public OR created_by = auth.uid() OR EXISTS(member)`

## Verified
Policy definition confirmed via pg_policies — all three conditions present.
